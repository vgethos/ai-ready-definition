April 24, 2026 — Validated against EDS v2 source

The Starting Point

Our foundations are shaky.

Experienced developers work around the gaps — they know which font is current, which prop name is the exception, which Storybook description to ignore. AI agents can’t. They follow the system literally, including its contradictions.

What’s missing

Text Styles
~90 classes (legacy + current) in one file. No separation. Agent can’t tell which are current.
Colors
Token system exists (253 tokens) but no enforcement of which tier to use. No guidance on what color to use when.
Spacing
10 tokens defined (2px–64px). Zero components use them. Entirely orphaned.
Fonts
Hacky implementation. Legacy (Cambon, NewTheinhardt) and current (Hauss, Portada) both shipped. Nothing enforces which to use.
Components
Poorly architected. Missing states. Inconsistent prop naming (size / inputSize / buttonSize). No layout templates.
Documentation & Guidelines
Very poor and incomplete. What exists is just property descriptions, not usage guidance. Storybook: “buttons come in different sizes to accommodate various use cases.” We need to start over.

Glossary

Tokens
Named variables for colors, spacing, and fonts. Change one value, it updates everywhere.
Color tokens work. Spacing tokens exist but no component uses them.
MCP
Model Context Protocol. How AI agents connect to external tools like Figma or a design system.
Without MCPs, an agent is blind to your designs and components.
Code Connect
Figma feature that maps design components to code. A small file links a Figma button to <Button variant="primary">.
We have zero. Agents guess which code matches which design.
Component Registry
Single file listing every component, its options, and valid values. Agent reads one file and knows the whole system.
registry.json (27 components) is our strongest AI-readiness asset.
Props & Variants
Props are options you pass to a component. Variants are predefined styles: primary, danger, outline.
Named variants = agent picks one. No variants = agent overrides manually = drift.
JSDoc
Inline code comments that explain what each prop does. The only documentation agents can read.
2 out of 40+ props have JSDoc. That’s effectively zero.
Storybook
Visual component catalog at a URL. Humans browse it; agents can’t.
Invisible to AI. Anything only in Storybook doesn’t exist for agents.
CI Check
Automated test that runs on every code change. Catches mistakes before they ship.
Button’s registry drifted (22 vs 14 variants). A CI check would have caught it.
Semantic Naming
Naming by purpose, not appearance. --theme-fg-default instead of #0f0f0f.
Our semantic color layer works. The challenge is enforcing agents use it over raw tokens.
Design Drift
When UI gradually diverges from standards. Wrong green here, odd spacing there.
AI is 10× faster than humans. Drift doesn’t accumulate — it scales.

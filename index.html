The Starting Point
Our design system has less than people assume.

Before evaluating AI-readiness, it’s important to be honest about where the foundations are. The design system has gaps that affect humans and AI agents equally. AI just makes these gaps impossible to ignore because agents can’t compensate the way experienced designers and developers do.

What’s missing
Text Styles
Definitions exist but are ambiguous. No clear guidance on when to use what, how, or why. Naming in Figma doesn’t match naming in code.
Colors
Missing functional-level colors. No abstraction layer for theming or modes. No guidance on what color to use when.
Spacing
No spacing system anywhere. Entirely assumed and left to individual judgment. No shared scale in Figma or code.
Tokens
What may appear like tokens in some cases, but not a real token system. Figma naming doesn’t match code. No shared abstraction.
Components
Poorly architected from a variant, layering, and props perspective. Missing states. Would need significant rework even with proper foundations.
Documentation
Very poor and incomplete. What exists is just basic property descriptions, not actual usage guidance. Storybook says things like “buttons come in different sizes to accommodate various use cases” — not helpful. We need to start over.

Because these foundations are incomplete, everything built on top inherits the problems.

What “AI-Ready” Means

For AI Agents

  1. One file with every component, its options, usage rules, and defaults — no digging through source code
  2. Consistent, predictable naming across all components
  3. Design values (colors, fonts, spacing) that are correct and reliable
  4. Layout templates for common pages — without them, agents improvise structure from scratch every time

For Designers

  1. Figma and code are a 1:1 match — same names, same properties, same structure
  2. No manual overrides — every visual state has a named variant

For Engineering

  1. The component registry is the contract — if it’s not there, agents can’t use it
  2. Component options are a stable API — renaming them breaks AI workflows
  3. Automated checks keep the registry in sync with actual code
Glossary
Tokens
Named values for colors, spacing, and fonts. Instead of “#0a5c4e” everywhere, you write “cypress-green” and update one place if it changes.
If agents use tokens, every screen is on-brand. If they hardcode, every screen is a liability.
MCP (Model Context Protocol)
How AI agents connect to external tools. Figma MCP lets an agent “see” designs. A Design System MCP gives it your component rules.
Without MCPs, an agent is blind to your designs and components.
Code Connect (Figma-to-Code Bridge)
A Figma feature that links design components to code. A small file says “this Figma button with Style=Primary = <Button variant="primary">.”
Without it, agents guess which code matches which design. With 25+ components, guessing fails.
Component Index
A single file listing every component, its options, and valid values. An agent reads one file and knows everything the system offers.
Without it, agents scan dozens of files to understand what’s available.
Props & Variants
Props are options you pass to a component (“make it large, green, with this label”). Variants are predefined styles: primary, danger, outline.
Named variants = agent picks one. No variants = agent manually overrides = drift.
JSDoc
Comments in the code that explain what each option does. Like a tooltip for developers and AI agents.
Agents can read code but not Storybook. Inline docs are the only guidance they see.
Storybook
A visual catalog showing every component in every state. Used by humans as reference. Lives at a URL.
Agents can’t browse websites. Storybook is invisible to them.
CI Check
An automated test that runs on every code change. Catches mistakes before they ship.
Our registry already drifted (22 vs 14 variants). Automation would have caught it instantly.
Semantic Naming
Naming by purpose, not appearance. “error-red” instead of “#c33.” If the palette changes, the name still works.
Semantic names are future-proof. Hardcoded values break when anything changes.
Design Drift
When the product’s UI gradually diverges from standards. Wrong green here, odd spacing there.
AI outputs 10× faster than humans. Drift doesn’t accumulate — it scales exponentially.
